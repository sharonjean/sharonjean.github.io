what about class?

We know that Ruby is an object-oriented programming language, so how does class come into the picture? It's good to mention that a class is an object. Using classes, you can create objects in Ruby which are equipped with powerful built-in methods. Yet the real power lies in being able to create new methods that can be used throughout each instance of class.

A class about none other than...coffee:

    class Coffee


      def initialize(temp, flavor)

        @temp = temp

        @flavor = flavor

      end


      def temperature

        @temp

      end


      def flavor

        @flavor

      end


      def choice_of_beverage

        puts "Here's your delicious #{@temp} #{@flavor}!"

      end


    end

initialize

Let's break down what's going on in this class so far. initialize is a special method you can use that will set an object's state when (and for each time) it is created. The great part is that Ruby has access to the values (the two arguments shown), as it has been saved by way of instance variables.

instance variables

An instance variable, denoted in the example with an @ symbol, is an object's variable. So it acts like a local variable, but is available for use between all methods in the class.

instantiation

To get the class going, it needs to be instantiated—which is to say that objects created are instances of the class. We can do this by calling the .new method on the class like so:

best_coffee = Coffee.new("iced", "almond latte")

output

puts best_coffee.choice_of_beverage
=> "Here's your delicious iced almond latte!"

The options we can apply to this class are limitless! We can apply the size of the cup, quantity of beverages, and even apply special directions—half decaf, extra hot, hold the whip, anyone?

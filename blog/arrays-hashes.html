Arrays vs. Hashes: When to use them

Arrays and hashes are collection classes which are used to store (and retrieve) data. Possibilities are endless, however, as arrays and hashes can actually coexist together. For example, you can turn an array into a hash. But, first things first.

Examples

A simple array of rainbow colors: colors = ["red", "orange", "yellow", "green", "blue", "indigo", "purple"]

And now, a hash of my favorite lipstick colors from different brands: fave_lipstick = {"Jefree Star" => "Unicorn Blood", "Urban Decay" = "F-Bomb", "BlackMoonCosmetics" => "Sanguis", "Stila" => "Aria", "Lush" => "Decisive", "ABH" => "Catnip"}

Retrieval

The point of storing these collections of data, is so you can retrieve them! Array: To retrive an item from an array, it's important to note that each item is stored in an index, starting with the number 0. So, if we were to retrieve from our colors array, we would call it like this: colors[0] - which will return "red", the first item in the array. Hash: To retrieve a value from a Hash's key/value pair, you would retrive it like this: fave_lipstick["ABH"] - this returns "Catnip", which is the value for key "ABH" (and a most beautiful shade of mauve)!

When and why: arrays

-you need to create an ordered list

When and why: hashes

-you need to map a value to an item -you have data that comes in pairs that you need to work with (key-value)

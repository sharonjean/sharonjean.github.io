JavaScript functions vs. Ruby classes!

We got to know the power of classes in Ruby last week, which discussed the ability to create objects, and perform funcationality on them by creating new methods (or using the powerful built-in methods provided by Ruby), which could be performed and reused any number of times.

Let's recap that classy stuff:

    class Coffee

      def initialize(temp, flavor)
        @temp = temp
        @flavor = flavor
      end

      def temperature
        @temp
      end

      def flavor
        @flavor
      end

      def choice_of_beverage
        puts "#{@temp} #{@flavor}"
      end

    end

So what does this look like in JavaScript?

      //Javascript uses function instead of class
      function Coffee(temp, flavor) { //this is used much like a Ruby @variable
        this.temp = temp;
        this.flavor = flavor;
      }

      Coffee.prototype.choiceOfBeverage = function() {
        return this.temp + " " + this.flavor;
      }

      var carey = new Coffee("iced", "almond latte");
      console.log(carey.choiceOfBeverage());

syntactical differences

Ruby's instance variable, denoted in the example with an @ symbol, is similar to the JavaScript method this. JavaScript also uses function instead of class.

metaobjects!

Both programs are object-oriented programs, designed to define other objects. This is the concept of metaobjects! This allows both languages to define any kind of real world object by using their respective magical power; "class" or "constructor function".
